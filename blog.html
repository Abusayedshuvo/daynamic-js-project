01. Discuss the scope of var, let, and const

In the javaScript have 3 way to declare a variable. 1. var, 2. let, 3. const. They have different behavior. Let's discuss this.

var: var is one way to declare a variable. This syntax is easy. Like var name = value; .
var is function scope. That means this variable works only inside the function where the declaration is made.

Also the var is hoisted like global function. Then it can access before declared in code. But it will initialized with undefined

let: let syntax is as like as var. But let is a Block scope. Let only work on a block where it declare. Outside it is not accessible.
Also let is not hoisted like var. It not accessible before initialized. And let variable can not re-declared. But possible to reassign value.

const: const is a block scope like as let. It not possible to redeclare and reassign. Also it is not hoisted.
But possible to modify array and object properties.

2. Tell us the use cases of null and undefined

In javascript null and undefined perform absence value. And both return falsely value. But they used to sometime different purpose.

null: is used for a variable or object that have no value. Also it can use for a placeholder, that will be assigned later. Sometime null use for check condition and comparisons.

undefined: indicates that a variable or object property has not been initialized or a function not returned any value.

If trying to an object that does not exit, it returns undefined
